# SsangBak 콘텐츠 작성 표준

Updated: 2026-07-13

이 문서는 SsangBak의 모든 신규 글 작성, 리라이트, 자동화 초안 생성에 적용하는 기준이다.
목표는 단순 검색 노출이 아니라 독자가 바로 판단하고 행동할 수 있는 정보, 구글 AI Overview에 인용될 수 있는 답변형 콘텐츠, 네이버/다음에서도 읽기 쉬운 모바일 중심 글을 만드는 것이다.

## 1. 핵심 원칙

- 사람 우선 콘텐츠를 최우선으로 한다.
- 기존 상위 글을 재조합하지 않고 SsangBak만의 정보 이득을 반드시 넣는다.
- 첫 100자 안에 메인 키워드를 자연스럽게 포함한다.
- 질문형 소제목 아래에는 1~2문장의 직접 답변을 먼저 쓴다.
- 모든 수치, 일정, 가격, 조건은 공식 출처 또는 확인 가능한 근거를 우선한다.
- 최신 정보는 발행일과 최종 업데이트일을 함께 관리한다.
- AI 문체가 아니라 실제 담당자가 독자에게 설명하는 말투로 쓴다.

## 2. 금지 표현

아래 표현은 원칙적으로 쓰지 않는다.

- 알아보겠습니다
- 다양한 측면에서
- 결론적으로
- 첫째로
- 둘째로
- 마지막으로
- ~에 대해 알아보는 시간을 가져보겠습니다
- 중요한 것은 ~입니다만
- 도움이 되셨길 바랍니다
- 무조건
- 100% 보장

대신 바로 답한다.

예: "2026 에너지바우처는 소득 기준과 세대원 특성 기준을 동시에 확인해야 신청 가능성이 높습니다."

## 3. 검색 의도별 글 구조

| 검색 의도 | 독자 상태 | 추천 구조 | CTA |
| --- | --- | --- | --- |
| 정보형 | 알고 싶다 | 핵심 답변, 기준 설명, 표, FAQ | 관련 글 또는 공식 자료 |
| 탐색형 | 찾아가고 싶다 | 공식 사이트, 경로, 로그인/인증 방법, 오류 해결 | 공식 바로가기 |
| 상업형 | 고르고 싶다 | 비교표, 장단점, 상황별 추천, 주의사항 | 비교/견적/확인 버튼 |
| 거래형 | 신청·구매·예매하고 싶다 | 대상, 기간, 준비물, 절차, CTA, 실패 방지 팁 | 신청/예매/조회 버튼 |

키워드가 섞여 있으면 독자의 가장 급한 행동을 기준으로 구조를 정한다.
예를 들어 "임영웅 콘서트 예매"는 정보형이 아니라 거래형과 예매형에 가깝다.

## 4. 7단계 작성 프로세스

1. 키워드 구성
   메인 키워드 1개, 보조 키워드 2~5개, 문장형 롱테일 키워드 2~4개를 정한다.

2. 검색 의도 판단
   정보형, 탐색형, 상업형, 거래형 중 하나를 정하고 글 구조를 맞춘다.

3. 차별화 포인트 설계
   상위 글에 없는 최신 일정, 공식 링크, 표, 지역별 차이, 실제 주의사항, 계산 예시, 체크리스트 중 최소 1개를 넣는다.

4. 목차 설계
   H2는 4~7개를 기본으로 하고, H3는 필요한 경우에만 0~3개 사용한다.

5. 본문 작성
   도입부에 핵심 답변을 먼저 배치하고, 문단은 2~4문장으로 짧게 구성한다.

6. 온페이지 최적화
   제목, 메타 설명, slug, 내부 링크, 이미지 ALT, CTA 위치를 점검한다.

7. 발행 후 리라이트 계획
   발행 4~8주 뒤 Search Console에서 노출은 있는데 클릭이 낮은 글, 11~20위권 글, 오래된 일정 글을 우선 보강한다.

아래 순서를 기본값으로 사용한다.
주제에 맞지 않는 섹션은 억지로 넣지 않는다.

1. 제목
2. SEO 메타 설명

4. 도입부 핵심 답변

## 6. 제목, 메타, slug 기준

- SEO 제목은 공백 포함 50~60자 안쪽을 우선한다.
- 클릭 유도는 하되 과장하지 않는다.
- 메타 설명은 140~160자 안쪽으로 작성한다.
- 메타 설명에는 메인 키워드, 핵심 혜택, 독자가 얻는 정보를 넣는다.
- slug는 짧은 영어 소문자로 작성하고, 불필요한 연도와 전치사는 줄인다.

예:

```txt
title: "2026 임영웅 콘서트 예매 일정 좌석 시야 총정리"
description: "2026 임영웅 콘서트 예매 일정, 티켓 오픈 시간, 공연장 좌석 시야, 구매 제한과 예매 성공 팁을 공식 정보 기준으로 정리했습니다."
slug: "lim-youngwoong-concert-ticket"
```

## 7. E-E-A-T 보강 규칙

- 가능하면 공식 출처, 공공기관, 예매처, 주최/주관 공지를 우선 인용한다.
- 수치가 들어가면 기준 날짜를 함께 적는다.
- 정책, 금융, 건강, 세금은 단정하지 않고 확인 경로를 제공한다.
- 작성자 또는 검수자 관점의 문구를 넣을 수 있다.
- 직접 확인한 화면, 계산 예시, 표 정리, 신청 흐름 등 1차 가공 정보를 넣는다.

권장 출처:

- 구글 검색 센터
- 정부24
- 복지로
- 홈택스
- 통계청
- 한국인터넷진흥원
- 정보통신정책연구원
- 공식 예매처 및 주최/주관 공지
- Semrush, Ahrefs, Pew Research Center 등 해외 리서치 기관

## 8. AI Overview 대응 규칙

- H2를 질문형으로 쓸 수 있으면 질문형으로 쓴다.
- 질문 바로 아래 첫 문장은 답변으로 시작한다.
- 40~80자 안팎의 짧은 정의형 답변을 포함한다.
- 표, 목록, FAQ를 사용해 문맥을 구조화한다.
- FAQPage 리치 결과 표시는 제한되더라도 페이지 이해를 돕는 FAQ 구조는 유지한다.
- 단계형 글은 HowTo 구조를 염두에 두고 순서를 분명히 쓴다.

예:

```md
## 2026 임영웅 콘서트 예매는 어디서 하나요?

2026 임영웅 콘서트는 NOL 티켓에서 단독 판매되며, 다른 경로의 티켓 판매는 공식 안내 기준으로 인정되지 않습니다.
```

## 9. 이미지와 ALT 기준

- 본문 이미지는 1~2장을 기본으로 한다.
- 이미지 ALT는 장식 문구가 아니라 내용을 설명한다.
- 썸네일 ALT는 키워드와 상황을 함께 넣는다.

예:

```txt
2026 임영웅 콘서트 IM HERO THE STADIUM 2 예매 안내 이미지
에너지바우처 신청 대상과 지원 금액을 비교한 표
```

## 10. 내부 링크와 CTA 기준

- 관련 글 2개 이상을 자연스럽게 연결한다.
- CTA는 독자의 행동 지점 앞뒤에 배치한다.
- CTA 문구는 짧고 명확하게 쓴다.

권장 CTA:

- 신청 대상 확인하기
- 공식 예매처 바로가기
- 환급금 조회하기
- 지원금 신청하기
- 좌석 시야 확인하기

## 11. FAQ 기준

- 모든 신규 글 하단에는 FAQ 5개를 기본으로 넣는다.
- 질문은 실제 검색자가 묻는 형태로 쓴다.
- 답변은 첫 문장에 결론을 담는다.
- 글 주제와 직접 관련 없는 일반 SEO FAQ는 넣지 않는다.

## 12. 발행 전 자체 점검

발행 전 아래 항목을 통과해야 한다.

| 항목 | 기준 |
| --- | --- |
| 정보 이득 | 상위 글 재조합이 아닌 고유 표, 체크리스트, 최신 확인 정보 중 1개 이상 |
| 검색 의도 | 정보형/탐색형/상업형/거래형 중 하나에 맞는 구조 |
| 첫 100자 | 메인 키워드 포함 |
| H2 구조 | 4~7개, 키워드와 연관어 자연 배치 |
| 메타 설명 | 140~160자 안쪽 |
| 문단 | 모바일 기준 2~4문장 |
| 표/목록 | 복잡한 정보는 표 또는 목록으로 정리 |
| 출처 | 공식 출처 또는 확인 경로 포함 |
| CTA | 주제에 맞는 행동 버튼 1~2개 |
| FAQ | 실제 질문 5개 |
| 금지 표현 | AI 상투어 제거 |
